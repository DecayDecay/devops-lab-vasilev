# Правила участия в проекте

## Как внести вклад

1. Форкните репозиторий
2. Создайте ветку для вашей функциональности (`git checkout -b feature/my-feature`)
3. Сделайте коммит изменений (`git commit -m "Add my feature"`)
4. Отправьте ветку в удалённый репозиторий (`git push origin feature/my-feature`)
5. Создайте Pull Request

## Правила оформления коммитов

- Используйте осмысленные сообщения коммитов на английском языке
- Начинайте сообщение с глагола в повелительном наклонении: `Add`, `Fix`, `Update`, `Remove`
- Пример: `Add CI pipeline configuration`

## Ветвление

- `main` — стабильная версия
- `develop` — ветка для разработки
- `feature/*` — ветки для новой функциональности
- `fix/*` — ветки для исправления ошибок

## Code Review

- Все изменения проходят через Pull Request
- Требуется минимум одно одобрение перед мержем
- Конфликты должны быть разрешены автором PR
